---
name: 资产提取架构理解错误
description: "逐集提取"应为每集独立串行 LLM 调用 + 程序预筛选，而非批量提交所有集数
type: feedback
---

实现串行资产提取时，误以为"逐集提取"是把所有集数一次提交给 LLM 再逐集输出结果。正确架构是：每集一次独立 LLM 调用，串行执行，且每次调用前用程序预筛选全量资产库（用剧本文本匹配缩小候选集），LLM 在单次调用中完成 extraction + relation judgment + confidence + variants + episode summary。

**Why:** 若 80 集每集约 6000 字，全量资产库到后期可能有上百个资产，若不筛选直接传入 token 会爆炸。而且串行的意义正在于每集完成后更新全局库，下一集才能感知到前面提取的新资产。

**How to apply:** 实现任何"逐集/分批处理"逻辑前，先向用户复述架构理解并确认：
- 每次是一个还是多个 LLM 调用？
- 上下文（前集摘要、候选资产）如何传递？
- 处理结果如何写回全局状态？
不要假设自己理解正确就直接实现。
