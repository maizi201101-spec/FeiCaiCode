# Tab3 右侧面板高度超出视口

**问题描述**：
Tab3 装配与生成页面，右侧参考图/资产图区域整体太高，超出浏览器可视区域，需要滚动才能看完。需约束整体布局高度在视口内，图片缩略图尺寸可以缩小，不要太占地方。

**触发场景**：
用户在 Tab3 页面查看右侧资产面板时，内容超出屏幕可视范围，必须滚动才能看到完整内容。

**教训/建议**：
1. 面板区域应使用 max-height + overflow-y: auto 约束在视口高度内，而非无限撑高
2. 缩略图尺寸应合理缩小，避免单张图片占用过多空间
3. dev-builder 在实现侧面板/资产面板时，应始终考虑视口约束，使用 calc(100vh - offset) 或类似方式限高
4. code-review 应检查面板类组件是否有高度约束，防止内容溢出视口
