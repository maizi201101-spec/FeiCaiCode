# 项目创建与数据管理问题

**问题描述**：
用户在使用项目创建功能时遇到多个问题：

1. **项目文件夹创建失败**：新建项目时，没有正确创建项目文件夹，文件直接放在根目录而非独立子目录中
2. **目录选择方式不便**：建议改用本地文件系统选择器来选择项目目录，或在页面中提供目录管理区，支持加载、切换、删除项目目录
3. **路径填写无响应**：当前通过填写项目路径创建文件夹的方式没有反应，功能不工作
4. **前端遗留数据问题**：页面上方"选择集数"显示不对（只显示4集），怀疑是前端写死的数据或遗留数据未清理
5. **分集功能错乱**：分集逻辑存在问题，分集结果混乱
6. **测试数据残留**：tmp 文件夹内的测试项目文件夹需要清除

**触发场景**：
用户尝试创建新项目时，发现项目结构不符合预期，功能异常。

**教训/建议**：
1. 项目创建流程需要验证文件夹创建逻辑，确保在正确位置创建独立项目目录
2. 考虑改进项目目录选择方式，使用系统原生目录选择器或页面内目录管理功能
3. 检查路径创建功能的实现，修复无响应问题
4. 清理前端写死的测试数据，改为动态读取
5. 修复分集逻辑，确保按照正确的规则进行分集
6. 在开发完成后清理 tmp 文件夹中的测试数据
