# 剧本管理页面设计缺失

**问题描述**：
1. 导入全集剧本后，缺少合理的分集处理页面设计
2. 剧本导入后放在资产页上面显示，位置不合理，不方便操作
3. Product-Spec.md 中定义了分集梗概功能（summary.txt），但前端实现时没有对应的预处理页面

**触发场景**：
用户测试第一轮，导入全集剧本后发现无法进行分集预处理操作

**涉及功能模块**：
- 模块二：剧本管理
- 模块三：资产管理（资产提取）

**相关 Spec 条目**：
- Product-Spec.md 模块二：导入剧本、剧本预览与编辑、按集分段导入
- Product-Spec.md 模块三：每集提取时同步输出该集梗概存为 summary.txt

**教训/建议**：
1. 剧本管理应该有独立页面，不应该和资产页混在一起
2. 需要设计分集梗概预处理的 UI 流程：导入全集 → AI 辅助分集 → 生成分集梗概 → 用户确认
3. dev-builder Skill 在开发剧本管理功能时，应明确页面布局和流程设计
4. design-brief-builder 在确定设计方向时，应覆盖剧本分集处理流程
