---
name: 改版时未清理旧代码
description: 重写函数/模块后遗留旧代码死代码，用户明确要求"别老遗留旧代码了"
type: feedback
---

在重写资产提取逻辑时，旧的实现代码没有被删除，导致死代码残留在文件中。用户明确反馈过"以后别老遗留旧代码了"。

**Why:** 遗留旧代码会造成混淆，增加文件体积，后续维护时容易误读旧实现为当前实现。

**How to apply:** 每次重写某个函数或模块时，完成后必须：
1. 搜索同文件中是否有被替换的旧函数/类/变量定义
2. 确认旧代码没有其他调用方后，直接删除
3. 不要留注释掉的旧代码，也不要留"备用"代码
改完验证编译通过即可，不需要保留旧代码作为回退。
